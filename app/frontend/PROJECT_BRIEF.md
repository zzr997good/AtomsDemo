# Atoms 多人共创平台 — 项目简要说明

## 一、实现思路与关键取舍

### 核心设计理念

本项目旨在演示一个「多人协作 + AI 辅助」的软件共创平台原型。核心交互模式为：

> 多位参与者（产品经理、开发者、设计师等）在同一个共创空间中，通过对话驱动 AI Agent 完成需求分析、架构设计、代码生成的全流程。

### 关键取舍

| 决策点 | 选择 | 原因 |
|--------|------|------|
| 后端架构 | localStorage 模拟共享数据库 | Demo 阶段优先展示交互流程，避免后端部署复杂度 |
| LLM 对话 | 关键词匹配 + 预设回复 | 快速验证 UI/UX 流程，无需 API Key 即可体验 |
| 多用户同步 | 同浏览器切换账号 | 降低演示门槛，一个人即可体验完整协作流程 |
| 代码生成 | Mock 静态代码片段 | 聚焦展示「从需求到代码」的流转过程 |
| 构建过程 | 定时器模拟进度 | 展示模块化构建的可视化效果 |

### 架构分层

```
┌─────────────────────────────────────────────┐
│  UI Layer (React + Tailwind + shadcn/ui)    │
├─────────────────────────────────────────────┤
│  State Layer (projectStore.ts - localStorage)│
├─────────────────────────────────────────────┤
│  Mock Engine (mockEngine.ts - 预设问答)      │
├─────────────────────────────────────────────┤
│  Auth Layer (auth.ts - Demo账号 + 注册)      │
└─────────────────────────────────────────────┘
```

## 二、当前完成程度

### ✅ 已完成（Demo 级别）

- **多账号系统**：5 个预设 Demo 账号 + 自定义注册，每人独立颜色标识
- **项目管理**：创建项目、项目列表、我的/参与的项目分类
- **邀请协作**：发送邀请 → 对方登录看到通知 → 接受/拒绝 → 加入项目
- **协作聊天**：群聊模式，消息带用户颜色和头像，模拟协作者自动回复
- **方案共编**：Markdown 方案文档可编辑，修改记录按用户颜色追踪
- **批注系统**：对方案特定章节添加评论/建议/修改批注
- **模块可视化**：架构模块图 + 构建进度动画 + 测试覆盖率
- **代码预览**：模拟生成的代码文件展示
- **应用预览**：构建完成后的 UI 预览

### ⚠️ 当前局限（因 Demo 性质）

- **LLM 对话**：使用简单的关键词匹配替代真实 LLM，回复为预设文本
- **代码生成**：静态 Mock 数据，非真实 AI 生成
- **数据同步**：仅限同一浏览器（localStorage），不支持跨设备
- **构建流程**：纯前端模拟，无真实编译/部署

## 三、后续规划与时间投入方向

### 🔥 最高优先级：LLM Workflow 框架

这是整体功能的核心引擎。如果继续投入时间，我会将最多精力放在这里：

```
LLM Workflow 框架设计方向：
├── Multi-Agent 协调层
│   ├── 需求分析 Agent（理解用户意图，拆解任务）
│   ├── 架构设计 Agent（技术选型，模块划分）
│   ├── 代码生成 Agent（按模块生成代码）
│   └── 测试/审查 Agent（代码审查，测试用例）
├── 上下文管理
│   ├── 长对话记忆（向量化存储 + 检索）
│   ├── 项目知识库（文档、代码、设计稿索引）
│   └── 多轮推理链路追踪
├── 工具调用层
│   ├── 代码执行沙箱
│   ├── 文件系统操作
│   ├── 外部 API 集成
│   └── 数据库操作
└── 质量保障
    ├── 输出校验（lint、type check、test）
    ├── 人工审核节点（关键决策需人确认）
    └── 回滚机制（出错时回退到上一稳定状态）
```

### 📊 Feature：数据挖掘

- 分析项目历史数据，识别常见需求模式
- 技术栈推荐（基于项目类型、团队规模）
- 代码质量趋势分析
- 协作效率指标（响应时间、迭代次数、完成率）

### 💰 Feature：项目收益初步评估

- 基于项目类型和规模的开发成本估算
- 市场同类产品对标分析
- ROI 预测模型（开发投入 vs 预期收益）
- 风险评估（技术风险、市场风险、时间风险）

### 📢 Feature：项目主动推广

- 自动生成项目介绍文案（针对不同平台：ProductHunt、Twitter、技术社区）
- SEO 内容自动生成
- 目标用户画像分析 + 推广渠道建议
- A/B 测试文案生成

## 四、技术栈

| 层级 | 当前 | 未来规划 |
|------|------|----------|
| 前端 | React + TypeScript + Tailwind + shadcn/ui | 不变 |
| 状态管理 | localStorage | Atoms Cloud / PostgreSQL |

## 五、测试指南 — 完整功能演示步骤

以下步骤可由一个人在同一浏览器中完成，模拟多人协作的完整流程。

---

### 第一阶段：账号与登录

#### 步骤 1：Demo 账号一键登录
1. 打开应用，进入登录页面
2. 默认显示「Demo 账号」标签页，可以看到 5 个预设账号：
   - 🟠 Alice (alice@demo.com)
   - 🔵 Bob (bob@demo.com)
   - 🟢 Charlie (charlie@demo.com)
   - 🟣 Diana (diana@demo.com)
   - 🩷 Eve (eve@demo.com)
3. 点击 **Alice** 的卡片，一键登录
4. ✅ 验证：成功跳转到项目列表页，右上角显示 Alice 的橙色头像

#### 步骤 2：自定义注册（可选）
1. 退出登录（点击右上角退出按钮）
2. 在登录页切换到「注册」标签
3. 填写昵称、邮箱、密码（≥4位），点击「注册并登录」
4. ✅ 验证：注册成功后自动登录，进入项目列表页

#### 步骤 3：自定义登录（可选）
1. 退出后切换到「登录」标签
2. 输入刚才注册的邮箱和密码
3. ✅ 验证：登录成功，进入项目列表

---

### 第二阶段：项目管理

#### 步骤 4：创建新项目
1. 以 Alice 登录后，在项目列表页点击「+ 新建项目」
2. 输入项目名称（如"电商小程序"）和描述（可选）
3. 点击「创建」
4. ✅ 验证：自动跳转到项目工作空间，顶部显示项目名称

#### 步骤 5：查看默认项目
1. 返回项目列表（点击左上角返回箭头）
2. 可以看到"我的项目"区域列出了刚创建的项目
3. 项目卡片显示协作人数徽标和状态标签
4. ✅ 验证：点击项目卡片可进入对应工作空间

---

### 第三阶段：AI 对话与构建流程

#### 步骤 6：需求确认阶段
1. 进入项目工作空间，左侧为聊天面板
2. 可以看到预设的对话（AI 已经在确认需求）
3. 在输入框输入「确认」或「开始」
4. ✅ 验证：AI 回复确认信息，右侧「方案」面板更新为 Markdown 方案文档

#### 步骤 7：触发架构设计
1. 输入包含「架构」或「设计」关键词的消息（如"看看架构设计"）
2. ✅ 验证：
   - 中间面板出现模块架构图（Auth、Task、Category、Notification 四个模块）
   - 顶部状态标签变为「架构设计」

#### 步骤 8：触发构建流程
1. 输入「开始构建」或「build」
2. ✅ 验证：
   - 顶部状态变为「构建中」（黄色脉冲动画）
   - 中间模块图逐个变色（灰→蓝→绿），显示构建进度
   - 右侧自动切换到「代码」标签，逐步出现生成的代码文件
   - 每个模块显示测试覆盖率百分比
   - 约 10 秒后全部完成，状态变为「已完成」（绿色）
   - 右侧「预览」标签显示模拟的 TaskFlow 应用 UI

---

### 第四阶段：方案编辑与修改追踪

#### 步骤 9：编辑方案文档
1. 切换到右侧「方案」标签
2. 点击方案文档区域（或右上角「编辑」按钮）
3. 进入编辑模式，修改 Markdown 内容（如添加一行"- 新增：暗色模式支持"）
4. 点击「保存」
5. ✅ 验证：方案内容更新，修改记录中出现 Alice 的橙色标记

#### 步骤 10：查看修改历史
1. 点击右上角的历史记录按钮（数字徽标）
2. ✅ 验证：底部展开修改记录面板，显示"Alice 修改了方案"及时间

---

### 第五阶段：多人协作（核心演示）

#### 步骤 11：Alice 邀请 Bob
1. 确保当前以 Alice 登录，在项目工作空间中
2. 点击顶部协作者栏的「+ 邀请」按钮
3. 弹出邀请面板，选择身份为「协作者」
4. 在用户列表中勾选 **Bob**
5. 点击「邀请」
6. ✅ 验证：
   - 邀请面板关闭
   - 约 2 秒后，聊天中出现 Bob 的蓝色消息"大家好！我加入了共创空间 👋"
   - 顶部协作者栏出现 Bob 的蓝色头像

#### 步骤 12：切换到 Bob 视角
1. 点击右上角退出登录
2. 在登录页选择 **Bob** 账号登录
3. ✅ 验证：
   - 项目列表页顶部出现黄色邀请通知卡片
   - 显示"Alice 邀请你加入项目「xxx」"
   - 有「接受」和「拒绝」按钮

#### 步骤 13：Bob 接受邀请
1. 点击「接受」按钮
2. ✅ 验证：
   - 邀请通知消失
   - 「参与的共创项目」区域出现该项目
   - 项目卡片显示 Alice 的橙色头像和"由 Alice 创建"

#### 步骤 14：Bob 在项目中协作
1. 点击项目进入工作空间
2. 可以看到之前 Alice 和 AI 的对话历史
3. Bob 在聊天中发送消息（如"我负责前端部分"）
4. ✅ 验证：
   - Bob 的消息以蓝色头像和蓝色标识显示
   - AI 可能回复（基于关键词匹配）
   - 其他在线协作者可能自动回复

#### 步骤 15：Bob 编辑方案
1. Bob 切换到「方案」标签，点击编辑
2. 修改内容（如添加"- 前端使用 React + TypeScript"）
3. 保存
4. ✅ 验证：修改历史中出现 Bob 的蓝色标记"Bob 修改了方案"

#### 步骤 16：切回 Alice 查看 Bob 的更新
1. 退出 Bob，重新以 Alice 登录
2. 进入同一个项目
3. ✅ 验证：
   - 聊天记录中可以看到 Bob 发送的蓝色消息
   - 方案标签顶部显示蓝色横幅"Bob 修改了方案"
   - 修改历史中有 Bob 的编辑记录

---

### 第六阶段：模块评论（可选）

#### 步骤 17：对模块添加评论
1. 在中间的模块架构图中，点击任意模块（如 Auth）
2. 在弹出的评论区输入评论内容
3. 提交评论
4. ✅ 验证：评论以当前用户的颜色显示在模块下方

---

### 第七阶段：多人同时邀请（可选）

#### 步骤 18：邀请多位协作者
1. 以 Alice 登录，打开邀请面板
2. 同时勾选 Charlie 和 Diana
3. 点击邀请
4. ✅ 验证：
   - 聊天中依次出现 Charlie（绿色）和 Diana（紫色）的加入消息
   - 协作者栏显示多个彩色头像
   - 后续对话中可能收到不同颜色协作者的自动回复

---

### 快速演示路径（3 分钟版）

如果时间有限，按以下最短路径演示核心功能：

1. Alice 登录 → 创建项目 → 输入"确认" → 输入"开始构建" → 观看构建动画
2. 邀请 Bob → 看到 Bob 加入消息
3. 退出 → Bob 登录 → 接受邀请 → 进入项目 → 发送消息 → 编辑方案
4. 退出 → Alice 登录 → 看到 Bob 的消息和方案修改

---

### 注意事项

- 所有数据存储在浏览器 localStorage 中，清除浏览器数据会重置所有状态
- 「开始构建」关键词触发构建动画，其他关键词如「架构」「设计」「确认」也有对应响应
- 协作者的自动回复是随机触发的（约 60% 概率），用于模拟真实协作氛围
- 同一浏览器中切换账号即可模拟不同用户，无需多个浏览器窗口
