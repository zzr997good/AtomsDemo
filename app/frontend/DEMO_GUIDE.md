# Atoms 多人共创 Demo 使用指南

## 概述

Atoms 多人共创 Demo 是一个展示 AI 驱动的协作开发平台的演示应用。多位用户可以同时登录不同的 Demo 账号，体验实时协作、对话式开发、模块可视化等功能。

---

## 快速开始

### 第一步：访问应用

在浏览器中打开部署后的应用链接（或本地开发地址 `http://localhost:5173`）。

### 第二步：选择 Demo 账号登录

进入登录页面后，你会看到 5 个可选的 Demo 账号：

| 账号 | 邮箱 | 角色名 |
|------|------|--------|
| 用户 1 | user1@demo.com | Alice |
| 用户 2 | user2@demo.com | Bob |
| 用户 3 | user3@demo.com | Charlie |
| 用户 4 | user4@demo.com | Diana |
| 用户 5 | user5@demo.com | Eve |

> ⚠️ **重要**：每位参与 Demo 的人应选择**不同的账号**登录，以体验多人协作效果。

点击任意账号即可直接登录，无需密码。

### 第三步：体验核心功能

登录后进入主界面，你可以体验以下功能：

#### 🗨️ 对话式开发（左侧面板）
- 在聊天框中输入需求描述（如"创建一个电商网站"）
- AI 会自动生成架构方案、代码文件
- 支持多轮对话，逐步完善项目

#### 🧩 模块可视化（中间区域）
- 查看项目的模块依赖关系图
- 点击模块节点查看详情
- 可对模块添加评论和标注

#### 📋 计划预览（右侧面板）
- 查看 AI 生成的项目计划（Markdown 格式）
- 支持对计划内容添加批注
- 实时同步所有协作者的标注

#### 👥 协作功能
- 顶部协作者栏显示当前在线用户
- 点击"邀请"按钮可邀请更多协作者
- 评论和标注会实时同步

### 第四步：退出登录

点击右上角的退出按钮即可退出，返回登录页面选择其他账号。

## 多人协作 Demo 演示流程

适用于向团队或客户演示多人协作功能：

### 准备工作
1. 准备 2-3 台设备（或浏览器窗口）
2. 每台设备打开应用链接

### 演示步骤

1. **设备 A** → 选择 `Alice (user1@demo.com)` 登录
2. **设备 B** → 选择 `Bob (user2@demo.com)` 登录
3. **设备 C**（可选）→ 选择 `Charlie (user3@demo.com)` 登录

4. 在设备 A 中输入对话消息，观察 AI 响应
5. 在设备 B 中对模块添加评论
6. 展示协作者栏中的在线状态
7. 展示计划批注的实时同步

### 演示要点
- 强调不同用户看到的是同一个项目
- 展示评论/批注的归属（不同用户名）
- 展示从对话到代码生成的完整流程

## 本地开发

```bash
# 安装依赖
pnpm install

# 启动开发服务器
pnpm run dev

# 构建生产版本
pnpm run build

# 代码检查
pnpm run lint
```

---

## 部署

### 方式一：通过 Atoms 平台发布
1. 在 App Viewer 中点击 **Publish** 按钮
2. 自定义 URL 路径
3. 点击发布，获取公开链接

### 方式二：自行部署
1. 运行 `pnpm run build` 生成 `dist/` 目录
2. 将 `dist/` 目录部署到任意静态托管服务（Vercel、Netlify、GitHub Pages 等）

## 技术栈

- **前端框架**: React 18 + TypeScript
- **UI 组件**: shadcn/ui + Tailwind CSS
- **路由**: React Router v6
- **构建工具**: Vite
- **后端**: Atoms Cloud（数据持久化）

## 常见问题

**Q: 不同用户的数据会互相影响吗？**
A: Demo 模式下，每个用户有独立的会话状态。协作功能通过模拟数据展示。

**Q: 如何重置 Demo 状态？**
A: 退出登录后重新登录即可获得初始状态，或清除浏览器 localStorage。

**Q: 可以同时在多个标签页登录不同账号吗？**
A: 不建议。同一浏览器共享 localStorage，请使用不同浏览器或隐私窗口模拟多用户。
