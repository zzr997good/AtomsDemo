# Project Summary
Atoms is a collaborative task management web application designed to enhance team productivity through efficient task management and real-time collaboration. It features a user-friendly interface that enables seamless communication and task tracking among team members. Users can explore the platform without registration via multiple demo login accounts. Recent updates have introduced a multi-user collaboration flow, allowing users to invite others to projects and view real-time updates, alongside detailed testing guidance for users.

# Project Module Description
- **Backend**: Manages data persistence, user authentication, and API interactions.
- **Frontend**: Provides the user interface for task management, collaboration features, and real-time updates.

# File Description Inventory
- **backend/README.md**: Documentation for backend setup and usage.
- **frontend/README.md**: Documentation for frontend setup and usage.
- **auth.ts**: Manages user authentication, including support for multiple demo login accounts.
- **Index.tsx**: Main page of the application, managing user interactions and data loading.
- **Login.tsx**: Handles user login functionality, supporting demo accounts and custom registration.
- **AppViewer.tsx**: Displays the application plan with real-time updates and collaboration features.
- **InvitePanel.tsx**: Manages user invitations for collaboration on projects.
- **Projects.tsx**: Displays the user's projects and handles project creation and invitations.
- **projectStore.ts**: Core data storage using localStorage for managing projects and user interactions.
- **mockEngine.ts**: Provides mock responses for testing and development purposes.
- **PROJECT_BRIEF.md**: Summary document outlining the project features, development roadmap, and detailed testing guidance for users.

# Technology Stack
- **Backend**: Python, Flask, SQLAlchemy, Alembic
- **Frontend**: React, TypeScript, Vite, Tailwind CSS
- **Database**: Atoms Cloud (NoSQL)

# Usage
To set up the project:
1. Install dependencies:
   - For the backend: `pip install -r requirements.txt`
   - For the frontend: `pnpm install`
2. Build the frontend application:
   - Run the build command for the frontend.
3. Start the backend server:
   - Execute the main.py file for the backend service.
